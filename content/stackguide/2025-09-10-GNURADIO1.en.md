## Available installation methods

GNU Radio can be installed in three main ways, each with its own trade-offs:

### 1. System package manager
**Advantages:**
- Fast and simple installation
- Automatic dependency management
- Optimal system integration
- Updates via system

**Disadvantages:**
- Often outdated version
- Limited compilation options
- Sometimes incomplete dependencies for certain modules

### 2. PyBOMBS (Python Build System)
**Advantages:**
- Recent versions available
- Third-party module management
- Isolated installation possible
- SDR-optimized configuration

**Disadvantages:**
- More complex to configure
- Long compilation time
- Sometimes unstable on certain distributions

### 3. Compilation from sources
**Advantages:**
- Latest version
- Full control of options
- Specific optimizations possible

**Disadvantages:**
- Very technical
- Manual dependency management
- Complex maintenance

## Installation via package manager (Ubuntu/Debian)

### Basic installation

```bash
# Package update
sudo apt update

# GNU Radio installation
sudo apt install gnuradio

# Complementary modules installation
sudo apt install gnuradio-dev
sudo apt install gr-osmosdr  # RTL-SDR, HackRF, etc. support
sudo apt install gr-fcdproplus  # FunCube Dongle Pro+ support
```

### Installation verification

```bash
# Check installed version
gnuradio-config-info --version

# Launch GNU Radio Companion
gnuradio-companion
```

**Typical test:** On Ubuntu 22.04 LTS, this method generally installs GNU Radio 3.10.1, stable but not the latest version.

## Installation via PyBOMBS

### Preparations

```bash
# PyBOMBS installation
pip3 install --user pybombs

# Add path if necessary
echo 'export PATH=$PATH:~/.local/bin' >> ~/.bashrc
source ~/.bashrc
```

### Configuration and installation

```bash
# Add official recipes
pybombs recipes add-defaults

# Create installation prefix
pybombs prefix init ~/gnuradio -a myprefix

# Install GNU Radio
pybombs -p myprefix install gnuradio
```

### Environment configuration

```bash
# Environment activation
source ~/gnuradio/setup_env.sh

# Add permanently to .bashrc
echo 'source ~/gnuradio/setup_env.sh' >> ~/.bashrc
```

**Typical duration:** 45-90 minutes depending on machine (full compilation).

## Installation from sources

### System prerequisites

```bash
# Compilation dependencies
sudo apt install git cmake g++ libboost-all-dev libgmp-dev 
sudo apt install swig python3-numpy python3-mako python3-sphinx 
sudo apt install python3-lxml doxygen libfftw3-dev libsdl1.2-dev 
sudo apt install libgsl-dev libqwt-qt5-dev libqt5opengl5-dev 
sudo apt install python3-pyqt5 liblog4cpp5-dev libzmq3-dev 
sudo apt install python3-yaml python3-click python3-click-plugins
```

### Compilation

```bash
# Source download
git clone --recursive https://github.com/gnuradio/gnuradio.git
cd gnuradio

# Configuration
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ../

# Compilation (uses all available cores)
make -j$(nproc)

# Installation
sudo make install
sudo ldconfig
```

**Compilation time:** 1-3 hours depending on machine power.

## Installation validation and testing

### Basic tests

```bash
# Python libraries test
python3 -c "import gnuradio; print('GNU Radio imported successfully')"

# Available modules verification
python3 -c "import gnuradio.blocks as blocks; print('Module blocks OK')"
python3 -c "import gnuradio.filter as filter; print('Module filter OK')"
```

### Test with simple flowgraph

Create a basic signal generator in GNU Radio Companion:
1. Signal Source → Throttle → QT GUI Time Sink
2. Parameters: frequency 1kHz, sampling 32kHz
3. Execute and verify display

### Performance test

```python
#!/usr/bin/env python3
import time
import numpy as np
from gnuradio import blocks, gr

class TestPerformance(gr.top_block):
    def __init__(self):
        gr.top_block.__init__(self)
        
        # Data source
        src = blocks.vector_source_c([1+1j]*1000000, True)
        
        # FIR filter (CPU load)
        taps = [1.0/64]*64
        fir = blocks.fir_filter_ccc(1, taps)
        
        # Null sink (no display)
        sink = blocks.null_sink(gr.sizeof_gr_complex)
        
        self.connect(src, fir, sink)

if __name__ == '__main__':
    tb = TestPerformance()
    start_time = time.time()
    tb.start()
    time.sleep(10)  # Test for 10 seconds
    tb.stop()
    tb.wait()
    elapsed = time.time() - start_time
    print(f"Test completed in {elapsed:.2f}s - Installation functional")
```

### Tested metrics

## Common problem resolution

### "module not found" error
```bash
# Check environment variables
echo $PYTHONPATH
echo $LD_LIBRARY_PATH

# For PyBOMBS, always source environment
source ~/gnuradio/setup_env.sh
```

### Permission problems
```bash
# Add user to dialout group (USB access)
sudo usermod -a -G dialout $USER
# Logout/login required
```

### GNU Radio Companion won't start
```bash
# Check Qt dependencies
sudo apt install python3-pyqt5 python3-pyqt5.qtopengl

# Test in debug mode
gnuradio-companion --log-level=DEBUG
```

## Recommendations by usage

### To get started quickly
**apt installation** - Simple and functional for learning.

### For serious usage
**PyBOMBS** - Good compromise between ease and functionality.

### For development
**Sources** - Full control and optimal performance.

## Conclusion

GNU Radio installation depends on your objectives. For a "stack guide" approach, we recommend PyBOMBS as the main method, with apt as a quick alternative for testing. Compilation from sources remains reserved for special cases requiring particular optimizations.
