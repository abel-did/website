Etter å ha installert GNU Radio, er det på tide å oppdage hovedgrensesnittet: GNU Radio Companion (GRC). Dette grensesnittet tillater å opprette signalbehandlingskjeder gjennom dra-og-slipp, og gjør GNU Radio-kraften tilgjengelig uten direkte programmering.

## GNU Radio Companion grensesnitt

### Oppstart og første inntrykk

```bash
# Start GNU Radio Companion
gnuradio-companion

# Eller fra Applikasjoner-menyen (avhengig av distribusjon)
# Applikasjoner → Engineering → GNU Radio Companion
```

GRC-grensesnittet består av fire hovedområder:

1. **Verktøylinje**: Ny, Åpne, Lagre, Utfør, Stopp
2. **Blokkepalett** (venstre): Katalog over tilgjengelige blokker, organisert etter kategori
3. **Sentralt lerret**: Arbeidsområde for å sette sammen flowgraph
4. **Konsoll** (bunn): Feilmeldinger, advarsler og programutgang

### Blokkepalettorganisasjon

Blokker er organisert i logiske kategorier:

- **Core**: Grunnleggende blokker (kilder, sinks, operatorer)
- **Filters**: FIR, IIR-filtre, desimatorer, interpolatorer  
- **Math Operations**: Matematiske operasjoner, FFT, korrelasjon
- **Modulators/Demodulators**: AM, FM, PSK, QAM, osv.
- **GUI**: Grafiske grensesnitt (scopes, spektra, kontroller)
- **Audio**: System audio-kilder og -sinks
- **Blocks**: Generiske og hjelpeverktøy-blokker

**Praktisk test**: Tell antall blokker tilgjengelig i din installasjon. En komplett GNU Radio 3.10+ installasjon inneholder over 200 grunnblokker.

## Grunnleggende konsepter: blokker og tilkoblinger

### Blokker: grunnenheter

Hver blokk i GNU Radio representerer en signalbehandlingsfunksjon:

**Sources**: Genererer data
- `Signal Source`: Genererer sinusoider, firkanter, trekanter
- `Noise Source`: Hvit, Gaussisk støy
- `File Source`: Leser datafiler

**Sinks**: Konsumerer data
- `QT GUI Time Sink`: Tidsdomene-oscilloskop
- `QT GUI Frequency Sink`: Spektrumanalysator
- `File Sink`: Skriver til fil

**Processing blocks**: Transformerer data
- `Multiply`: Signalmultiplikasjon
- `Add`: Signaladdisjon
- `Low Pass Filter`: Lavpassfilter

### Tilkoblingstyper

GNU Radio bruker et strengt typesystem:

- **Complex** (blå): Komplekse I/Q-data
- **Float** (oransje): Reelle flyttall
- **Int** (gul): Heltall
- **Byte** (grønn): 8-bits data

**Viktig regel**: Du kan bare koble porter av samme type. GRC indikerer typefeil med røde tilkoblinger.

## Første enkle flowgraph

### Grunnleggende signalgenerator

La oss opprette en første flowgraph for å generere og visualisere et signal:

1. **Legge til blokker**:
   - Dra `Signal Source` fra Core → Sources
   - Legg til `Throttle` fra Core → Misc  
   - Legg til `QT GUI Time Sink` fra Core → GUI

2. **Parameterkonfigurasjon**:

   **Signal Source**:
   - `Output Type`: Complex
   - `Sample Rate`: 32000
   - `Waveform`: Cosine
   - `Frequency`: 1000
   - `Amplitude`: 1

   **Throttle**:
   - `Type`: Complex
   - `Sample Rate`: 32000

   **QT GUI Time Sink**:
   - `Type`: Complex
   - `Sample Rate`: 32000
   - `Number of Points`: 1024

3. **Tilkoblinger**:
   ```
   Signal Source → Throttle → QT GUI Time Sink
   ```

### Flowgraph-analyse

Denne enkle flowgraph illustrerer nøkkelkonsepter:

- **Signal Source** genererer en kompleks sinusoide ved 1 kHz
- **Throttle** begrenser gjennomstrømning for å unngå CPU-overbelastning
- **Time Sink** viser signalet i sanntid

**Hvorfor Throttle?** Uten den ville GNU Radio generere data så raskt som mulig, og mettet CPU-en. Throttle simulerer en realistisk samplingsfrekvens.

## Datatyper og samplingsfrekvenser

### Typekorrespondanse

| GRC Type | Beskrivelse | Størrelse | Typisk bruk |
|----------|-------------|-----------|-------------|
| complex | Kompleks I/Q | 8 bytes | RF-signaler, grunnbånd |
| float | 32-bits float | 4 bytes | Lyd, reelle signaler |
| int | 32-bits heltall | 4 bytes | Tellere, kontroll |
| byte | 8-bits heltall | 1 byte | Binære data |

### Samplingsfrekvenshåndtering

**Gylden regel**: Alle blokker i en bane må ha samme samplingsfrekvens, unntatt når eksplisitt bruk av desimatorer/interpolatorer.

**Gjennomstrømningsberegningseksempel**:
- Komplekst signal ved 2 MSps = 2×10⁶ × 8 bytes = 16 MB/s
- Float signal ved 48 kSps = 48×10³ × 4 bytes = 192 kB/s

### Praktisk test: gjennomstrømningsfeiloppdagelse

Opprett en tilsiktet feil:
1. Signal Source (32 kSps) → Time Sink (48 kSps)
2. Utfør flowgraph
3. Observer feilen i konsollen:
   ```
   RuntimeError: check topology failed: input multiple different sample rates
   ```

## Utførelse og feilsøking

### Utførelsesmodi

GRC tilbyr flere utførelsesmodi:

**Normal modus** (F5):
- Genererer Python-kode
- Kompilerer hvis nødvendig  
- Starter utførelse

**Steg-for-steg modus**:
- `Generate` (F7): Genererer kun kode
- `Execute` (F6): Starter generert kode

### Vanlige feilmeldinger

**1. Tilkoblingsfeil**:
```
RuntimeError: cannot connect output port X to input port Y
```
*Løsning*: Sjekk datatyper (tilkoblingsfarger)

**2. Manglende parameter**:
```
Missing required parameter: Sample Rate
```
*Løsning*: Fyll ut alle obligatoriske felt (i rødt)

**3. Ikke-tilkoblet flowgraph**:
```
Flow graph has unconnected blocks
```
*Løsning*: Alle blokker må være tilkoblet i en sammenhengende bane

### Feilsøkingskonsoll

Konsollen viser nyttig informasjon:
- **INFO**: Generell utførelsesinformasjon
- **WARNING**: Ikke-blokkerende advarsler  
- **ERROR**: Feil som forhindrer utførelse
- **DEBUG**: Detaljert informasjon (verbose modus)

## Python-kodegenerering

### Automatisk generert kode

GRC genererer automatisk ekvivalent Python-kode. For en enkel flowgraph:

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-

from gnuradio import blocks
from gnuradio import gr
from gnuradio import qtgui
from gnuradio.filter import firdes

class simple_flowgraph(gr.top_block):
    def __init__(self):
        gr.top_block.__init__(self, "Simple Signal Generator")
        
        # Blocks
        self.signal_source = blocks.signal_source_c(
            32000,          # Sample rate
            blocks.GR_COS_WAVE,  # Waveform
            1000,           # Frequency
            1,              # Amplitude
            0               # Offset
        )
        
        self.throttle = blocks.throttle(
            gr.sizeof_gr_complex*1, 
            32000,
            True
        )
        
        self.qtgui_time_sink = qtgui.time_sink_c(
            1024,           # Size
            32000,          # Sample rate
            "",             # Name
            1               # Number of inputs
        )
        
        # Connections
        self.connect((self.signal_source, 0), (self.throttle, 0))
        self.connect((self.throttle, 0), (self.qtgui_time_sink, 0))
```

## Sammenligning med andre visuelle verktøy

### GNU Radio Companion vs alternativer

| Verktøy | Type | Fordeler | Ulemper |
|---------|------|----------|---------|
| **GRC** | Åpen kildekode | Gratis, utvidbar, aktiv fellesskap | Læringskurve |
| **MATLAB Simulink** | Proprietær | Polert grensesnitt, MATLAB-integrasjon | Kostnad, lock-in |
| **LabVIEW** | Proprietær | Utmerket for instrumentering | Svært dyrt, spesialisert |
| **SDR# Plugins** | Gratis/begrenset | Enkelt for mottak | Begrenset, kun Windows |

### GRC spesifikke styrker

1. **Komplett økosystem**: Over 200 grunnblokker
2. **Åpen kildekode**: Mulighet til å modifisere/utvide
3. **Multi-plattform**: Linux, Windows, macOS
4. **Kodegenerering**: Enkel overgang til ren Python
5. **Aktivt fellesskap**: Tredjepartsblokker, eksempler, støtte

### Optimale brukstilfeller

**GRC utmerker seg for**:
- Rask prototyping av SDR-algoritmer
- Utdanning og opplæring
- Konsepttesting før implementering
- Enkle/mellomstore sanntidsapplikasjoner

**GRC-begrensninger**:
- Begrenset ytelse for kritiske applikasjoner
- Noen ganger tregt grensesnitt med komplekse flowgraphs
- Vanskelig feilsøking på store prosjekter

## Konklusjon

GNU Radio Companion gir en tilgjengelig introduksjon til SDR-verdenen. Dens visuelle tilnærming tillater raskt å forstå dataflyt og signalbehandlingskonsepter. Selv om begrenset for produksjonsapplikasjoner, forblir GRC et utmerket prototyping- og læringsverktøy.
