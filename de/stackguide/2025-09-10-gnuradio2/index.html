      Erster Kontakt mit GNU Radio Companion

      Nach der Installation von GNU Radio ist es Zeit, die wichtigste grafische Benutzeroberfläche kennenzulernen: GNU Radio Companion (GRC). Diese Schnittstelle ermöglicht es, Signalverarbeitungsketten durch Drag & Drop zu erstellen und macht so die Macht von GNU Radio zugänglich, ohne direkt programmieren zu müssen.

      Benutzeroberfläche von GNU Radio Companion

      Start und erster Eindruck 

      # GNU Radio Companion starten
gnuradio-companion

# Oder über das Anwendungsmenü (je nach Distribution)
# Anwendungen → Engineering → GNU Radio Companion

      Die GRC-Benutzeroberfläche besteht aus vier Hauptbereichen:

      1. Symbolleiste: Neu, Öffnen, Speichern, Ausführen, Stoppen
      2. Block-Palette (links): Katalog der verfügbaren Blöcke, organisiert nach Kategorien
      3. Zentrale Leinwand: Arbeitsbereich zum Zusammenstellen des Flowgraphs
      4. Konsole (unten): Fehlermeldungen, Warnungen und Programmausgabe

      Organisation der Block-Palette

      Die Blöcke sind in logische Kategorien organisiert:

      • Core: Grundlegende Blöcke (Quellen, Senken, Operatoren)
      • Filters: FIR-, IIR-Filter, Dezimatoren, Interpolatoren
      • Math Operations: Mathematische Operationen, FFT, Korrelation
      • Modulators/Demodulators: AM, FM, PSK, QAM, etc.
      • GUI: Grafische Benutzeroberflächen (Scopes, Spektren, Steuerelemente)
      • Audio: Audio-Quellen und -Senken des Systems
      • Blocks: Generische und Utility-Blöcke

      Praktischer Test: Zählen Sie die Anzahl der verfügbaren Blöcke in Ihrer Installation. Eine vollständige GNU Radio 3.10+ Installation enthält mehr als 200 Grundblöcke.

      Grundkonzepte: Blöcke und Verbindungen

      Die Blöcke: Grundeinheiten

      Jeder Block in GNU Radio repräsentiert eine Signalverarbeitungsfunktion:

      Quellen: Generieren Daten

      • Signal Source: Generiert Sinuswellen, Rechteck-, Dreieckwellen
      • Noise Source: Weißes Rauschen, Gauß’sches Rauschen
      • File Source: Liest Datendateien

      Senken: Verbrauchen Daten

      • QT GUI Time Sink: Zeitbereichs-Oszilloskop
      • QT GUI Frequency Sink: Spektrumanalysator
      • File Sink: Schreibt in eine Datei

      Verarbeitungsblöcke: Transformieren Daten

      • Multiply: Multiplikation von Signalen
      • Add: Addition von Signalen
      • Low Pass Filter: Tiefpassfilter

      Verbindungstypen

      GNU Radio verwendet ein strenges Typsystem:

      • Complex (blau): Komplexe I/Q-Daten
      • Float (orange): Reelle Gleitkommazahlen
      • Int (gelb): Ganze Zahlen
      • Byte (grün): 8-Bit-Daten

      Wichtige Regel: Sie können nur Ports desselben Typs verbinden. GRC zeigt Typfehler durch rote Verbindungen an.

      Erster einfacher Flowgraph

      Einfacher Signalgenerator

      Erstellen wir einen ersten Flowgraph zum Generieren und Visualisieren eines Signals:

      1. Hinzufügen der Blöcke:

        • Ziehen Sie Signal Source aus Core → Sources
        • Fügen Sie Throttle aus Core → Misc hinzu
        • Fügen Sie QT GUI Time Sink aus Core → GUI hinzu

      2. Konfiguration der Parameter:

        Signal Source:

        • Output Type: Complex
        • Sample Rate: 32000
        • Waveform: Cosine
        • Frequency: 1000
        • Amplitude: 1

        Throttle:

        • Type: Complex
        • Sample Rate: 32000

        QT GUI Time Sink:

        • Type: Complex
        • Sample Rate: 32000
        • Number of Points: 1024

      3. Verbindungen:

        Signal Source → Throttle → QT GUI Time Sink

      Analyse des Flowgraphs

      Dieser einfache Flowgraph illustriert die Schlüsselkonzepte:

      • Signal Source generiert eine komplexe Sinuswelle bei 1 kHz
      • Throttle begrenzt den Durchsatz zur Vermeidung von CPU-Überlastung
      • Time Sink zeigt das Signal in Echtzeit an

      Warum der Throttle? Ohne ihn würde GNU Radio Daten so schnell wie möglich generieren und die CPU überlasten. Der Throttle simuliert eine realistische Abtastrate.

      Datentypen und Abtastraten

      Typenentsprechung

      GRC-Typ Beschreibung Größe Typische Verwendung
      complex Komplex I/Q 8 Bytes RF-Signale, Basisband
      float 32-Bit Gleitkomma 4 Bytes Audio, reelle Signale
      int 32-Bit Ganzzahl 4 Bytes Zähler, Steuerung
      byte 8-Bit Ganzzahl 1 Byte Binäre Daten

      Verwaltung der Abtastraten

      Goldene Regel: Alle Blöcke in einem Pfad müssen dieselbe Abtastrate haben, außer bei expliziter Verwendung von Dezimatoren/Interpolatoren.

      Beispiel einer Durchsatzberechnung:

      • Komplexes Signal bei 2 MSps = 2×10⁶ × 8 Bytes = 16 MB/s
      • Float-Signal bei 48 kSps = 48×10³ × 4 Bytes = 192 kB/s

      Praktischer Test: Durchsatzfehler-Erkennung

      Erstellen Sie absichtlich einen Fehler:

      1. Signal Source (32 kSps) → Time Sink (48 kSps)
      2. Führen Sie den Flowgraph aus
      3. Beobachten Sie den Fehler in der Konsole: 
        RuntimeError: check topology failed: input multiple different sample rates

      Ausführung und Debugging

      Ausführungsmodi

      GRC bietet verschiedene Ausführungsmodi:

      Normaler Modus (F5):

      • Generiert Python-Code
      • Kompiliert falls nötig
      • Startet die Ausführung

      Schrittweiser Modus:

      • Generate (F7): Generiert nur den Code
      • Execute (F6): Startet den generierten Code

      Häufige Fehlermeldungen

      1. Verbindungsfehler:

      RuntimeError: cannot connect output port X to input port Y

      Lösung: Überprüfen Sie die Datentypen (Farben der Verbindungen)

      2. Fehlender Parameter:

      Missing required parameter: Sample Rate

      Lösung: Füllen Sie alle Pflichtfelder (rot markiert) aus

      3. Nicht verbundener Flowgraph:

      Flow graph has unconnected blocks

      Lösung: Alle Blöcke müssen in einem kohärenten Pfad verbunden sein

      Debug-Konsole

      Die Konsole zeigt nützliche Informationen:

      • INFO: Allgemeine Informationen zur Ausführung
      • WARNING: Nicht blockierende Warnungen
      • ERROR: Fehler, die die Ausführung verhindern
      • DEBUG: Detaillierte Informationen (Verbose-Modus)

      Python-Code-Generierung

      Automatisch generierter Code

      GRC generiert automatisch äquivalenten Python-Code. Für einen einfachen Flowgraph:

      #!/usr/bin/env python3
# -*- coding: utf-8 -*-

from gnuradio import blocks
from gnuradio import gr
from gnuradio import qtgui
from gnuradio.filter import firdes

class simple_flowgraph(gr.top_block):
    def __init__(self):
        gr.top_block.__init__(self, "Simple Signal Generator")
        
        # Blocks
        self.signal_source = blocks.signal_source_c(
            32000,          # Sample rate
            blocks.GR_COS_WAVE,  # Waveform
            1000,           # Frequency
            1,              # Amplitude
            0               # Offset
        )
        
        self.throttle = blocks.throttle(
            gr.sizeof_gr_complex*1, 
            32000,
            True
        )
        
        self.qtgui_time_sink = qtgui.time_sink_c(
            1024,           # Size
            32000,          # Sample rate
            "",             # Name
            1               # Number of inputs
        )
        
        # Connections
        self.connect((self.signal_source, 0), (self.throttle, 0))
        self.connect((self.throttle, 0), (self.qtgui_time_sink, 0))

      Vorteile des visuellen Ansatzes

      Für das Lernen:

      • Sofortige Visualisierung des Datenflusses
      • Parameter über grafische Benutzeroberfläche zugänglich
      • Echtzeit-Fehlererkennung

      Für das Prototyping:

      • Schnelle Parameteränderung
      • Test verschiedener Konfigurationen
      • Speicherung von Varianten

      Vergleich mit anderen visuellen Tools

      GNU Radio Companion vs. Alternativen

      Tool Typ Vorteile Nachteile
      GRC Open Source Kostenlos, erweiterbar, aktive Community Lernkurve
      MATLAB Simulink Proprietär Gepflegte Benutzeroberfläche, MATLAB-Integration Kosten, Lock-in
      LabVIEW Proprietär Exzellent für Instrumentierung Sehr teuer, spezialisiert
      SDR# Plugins Kostenlos/begrenzt Einfach für Empfang Begrenzt, nur Windows

      Spezifische Stärken von GRC

      1. Vollständiges Ökosystem: Mehr als 200 Grundblöcke
      2. Open Source: Möglichkeit zur Modifikation/Erweiterung
      3. Multiplattform: Linux, Windows, macOS
      4. Code-Generierung: Einfacher Übergang zu reinem Python
      5. Aktive Community: Drittanbieter-Blöcke, Beispiele, Support

      Optimale Anwendungsfälle

      GRC excellt bei:

      • Schnelles Prototyping von SDR-Algorithmen
      • Bildung und Ausbildung
      • Konzepttests vor Implementierung
      • Einfache/mittlere Echtzeitanwendungen

      GRC-Grenzen:

      • Begrenzte Performance für kritische Anwendungen
      • Benutzeroberfläche manchmal langsam bei komplexen Flowgraphs
      • Schwieriges Debugging bei großen Projekten

      Praktische Übungen

      Übung 1: Multi-Frequenz-Signal

      Erstellen Sie einen Generator für zwei Sinuswellen (1 kHz und 3 kHz):

      1. Zwei Signal Source
      2. Ein Add-Block zum Kombinieren
      3. Visualisierung auf Time Sink und Frequency Sink

      Übung 2: Spektralanalyse

      Fügen Sie einen Spektrumanalysator zu Ihrem ersten Flowgraph hinzu:

      1. Verbinden Sie den Throttle mit QT GUI Frequency Sink
      2. Beobachten Sie den Peak bei 1 kHz
      3. Ändern Sie die Generatorfrequenz, beobachten Sie die Änderung

      Übung 3: Einfache Modulation

      Implementieren Sie eine grundlegende Amplitudenmodulation:

      1. Trägersignal (10 kHz)
      2. Modulierendes Signal (1 kHz)
      3. Multiplizierer für die Modulation
      4. Zeit- und Frequenzbereichsvisualisierung

      Fazit

      GNU Radio Companion bietet eine zugängliche Einführung in die Welt der SDR. Sein visueller Ansatz ermöglicht es, schnell die Konzepte von Datenfluss und Signalverarbeitung zu verstehen. Obwohl für Produktionsanwendungen begrenzt, bleibt GRC ein exzellentes Tool für Prototyping und Lernen.

      Im nächsten Artikel werden wir diese Grundlagen nutzen, um unseren ersten vollständigen AM-Modulator zu erstellen und dabei fortgeschrittenere Konzepte der Modulation und Signalgenerierung einzuführen.
