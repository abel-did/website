      Creating Your Own GNU Radio Blocks

      After mastering installation, the GRC interface, and creating complex flowgraphs, it’s time to take the next step: creating your own custom blocks. This skill transforms GNU Radio from a prototyping tool into a complete development platform.

      Types of Blocks in GNU Radio

      Classification by Functionality

      GNU Radio organizes blocks into several categories based on their behavior:

      Sync Blocks: Fixed 1:1 input/output ratio

      • One input sample → One output sample
      • Example: Multiply, Add, Complex to Mag
      • Simplest to implement

      General Blocks: Variable input/output ratio

      • Number of consumed/produced samples can vary
      • Example: Decimator, Interpolator, Frame Sync
      • More complex but very powerful

      Hierarchical Blocks: Composition of existing blocks

      • Encapsulates multiple blocks into one
      • Reusability and interface simplicity
      • Example: Complete modulator, Composite filter

      Source/Sink Blocks: Interface with the outside world

      • Sources: Generate data
      • Sinks: Consume data
      • Hardware interface or files

      Choosing Type by Use Case

      Type Use Case Complexity Performance
      Sync Simple mathematical operations Low High
      General Rate change, protocols High Variable
      Hierarchical Reuse, simple interface Medium Inherited
      Source/Sink Hardware interface High Critical

      When to Create a Custom Block

      Warning Signs for a Custom Block

      Overly complex flowgraph:

      • More than 20 blocks for a simple function
      • Repetition of the same subset of blocks
      • Difficulty maintaining or modifying

      Insufficient performance:

      • Repetitive operations in pure Python
      • Identified bottlenecks
      • Latency too high for the application

      Missing functionality:

      • Proprietary or specialized algorithm
      • Interface with specific hardware
      • Unsupported communication protocol

      Alternatives to Consider First

      Before creating a block, check:

      1. Existing gr- modules*: Over 50 third-party modules available
      2. Block combination: Hierarchical block might suffice
      3. External Python scripts: File Source/Sink for interface

      Creating a Simple Python Block

      Example: Adaptive Peak Detector

      Let’s create a block that detects peaks in a signal with an adaptive threshold.

      Step 1: Basic Structure

      Create a new file peak_detector.py:

      #!/usr/bin/env python3

import numpy as np
from gnuradio import gr

class peak_detector(gr.sync_block):
    """
    Adaptive peak detector
    """
    def __init__(self, threshold_ratio=0.8, adaptation_rate=0.01):
        # Input/output port definition
        gr.sync_block.__init__(
            self,
            name="peak_detector",
            in_sig=[np.float32],    # One float input
            out_sig=[np.float32]    # One float output
        )
        
        # Block parameters
        self.threshold_ratio = threshold_ratio
        self.adaptation_rate = adaptation_rate
        
        # Internal state variables
        self.peak_level = 0.0
        self.threshold = 0.0

    def work(self, input_items, output_items):
        """
        Main processing function
        """
        in0 = input_items[0]    # Input signal
        out = output_items[0]   # Output signal
        
        # Sample-by-sample processing
        for i in range(len(in0)):
            sample = abs(in0[i])
            
            # Threshold adaptation
            if sample > self.peak_level:
                self.peak_level = sample
            else:
                # Slow peak decay
                self.peak_level *= (1 - self.adaptation_rate)
            
            # Threshold calculation
            self.threshold = self.peak_level * self.threshold_ratio
            
            # Peak detection
            if sample > self.threshold:
                out[i] = 1.0  # Peak detected
            else:
                out[i] = 0.0  # No peak
        
        # Return number of processed samples
        return len(output_items[0])

      Step 3: Unit Testing the Block

      Create test_peak_detector.py:

      #!/usr/bin/env python3

import numpy as np
import matplotlib.pyplot as plt
from peak_detector import peak_detector

def test_peak_detector():
    # Generate test signal
    t = np.linspace(0, 1, 1000)
    signal = np.sin(2 * np.pi * 5 * t)  # 5 Hz
    
    # Add artificial peaks
    signal[200:210] += 2.0  # Peak at t=0.2s
    signal[500:510] += 1.5  # Peak at t=0.5s
    signal[800:820] += 3.0  # Wider peak at t=0.8s
    
    # Block instantiation
    detector = peak_detector(threshold_ratio=0.7, adaptation_rate=0.05)
    
    # Processing simulation
    output = np.zeros_like(signal)
    
    # Block processing (GNU Radio simulation)
    block_size = 100
    for i in range(0, len(signal), block_size):
        end = min(i + block_size, len(signal))
        input_block = [signal[i:end]]
        output_block = [output[i:end]]
        
        detector.work(input_block, output_block)
    
    # Visualization
    plt.figure(figsize=(12, 6))
    plt.subplot(2, 1, 1)
    plt.plot(t, signal)
    plt.title('Input Signal with Peaks')
    plt.ylabel('Amplitude')
    
    plt.subplot(2, 1, 2)
    plt.plot(t, output)
    plt.title('Peak Detector Output')
    plt.ylabel('Detection (1=peak)')
    plt.xlabel('Time (s)')
    
    plt.tight_layout()
    plt.show()
    
    print(f"Peaks detected: {np.sum(output)} samples")

if __name__ == '__main__':
    test_peak_detector()

      Optimization and C++ Blocks

      When to Switch to C++

      Performance indicators:

      • Block consumes >50% of calculation time
      • Intensive mathematical operations
      • Loops over large data volumes
      • Critical latency

      Example C++ Port

      Basic structure for the same peak detector:

      // peak_detector_impl.h
#include <gnuradio/sync_block.h>

class peak_detector_impl : public gr::sync_block
{
private:
    float d_threshold_ratio;
    float d_adaptation_rate;
    float d_peak_level;
    float d_threshold;

public:
    peak_detector_impl(float threshold_ratio, float adaptation_rate);
    ~peak_detector_impl();

    int work(int noutput_items,
             gr_vector_const_void_star &input_items,
             gr_vector_void_star &output_items) override;

    void set_threshold_ratio(float threshold_ratio);
    float threshold_ratio() const;
};
      // peak_detector_impl.cc
int peak_detector_impl::work(int noutput_items,
                           gr_vector_const_void_star &input_items,
                           gr_vector_void_star &output_items)
{
    const float *in = (const float *) input_items[0];
    float *out = (float *) output_items[0];

    // Optimized loop
    for (int i = 0; i < noutput_items; i++) {
        float sample = std::abs(in[i]);
        
        // Vectorized adaptation if possible
        if (sample > d_peak_level) {
            d_peak_level = sample;
        } else {
            d_peak_level *= (1.0f - d_adaptation_rate);
        }
        
        d_threshold = d_peak_level * d_threshold_ratio;
        out[i] = (sample > d_threshold) ? 1.0f : 0.0f;
    }

    return noutput_items;
}

      Testing and Validation

      Complete Test Suite 

      #!/usr/bin/env python3

import unittest
import numpy as np
from peak_detector import peak_detector

class TestPeakDetector(unittest.TestCase):
    
    def setUp(self):
        self.detector = peak_detector(threshold_ratio=0.8, adaptation_rate=0.1)
    
    def test_no_input(self):
        """Test with null signal"""
        input_data = [np.zeros(100)]
        output_data = [np.zeros(100)]
        
        result = self.detector.work(input_data, output_data)
        
        self.assertEqual(result, 100)
        self.assertTrue(np.all(output_data[0] == 0))
    
    def test_constant_signal(self):
        """Test with constant signal"""
        input_data = [np.ones(100)]
        output_data = [np.zeros(100)]
        
        self.detector.work(input_data, output_data)
        
        # First sample should be detected
        self.assertEqual(output_data[0][0], 1.0)
    
    def test_parameter_bounds(self):
        """Test parameter limits"""
        # Test threshold_ratio
        self.detector.set_threshold_ratio(0.5)
        self.assertEqual(self.detector.threshold_ratio(), 0.5)
        
        # Test adaptation_rate  
        self.detector.set_adaptation_rate(0.01)
        self.assertEqual(self.detector.adaptation_rate(), 0.01)
    
    def test_peak_detection_accuracy(self):
        """Test detection accuracy"""
        # Signal with known peak
        signal = np.zeros(1000)
        signal[500:510] = 2.0  # 10-sample peak
        
        input_data = [signal]
        output_data = [np.zeros(1000)]
        
        self.detector.work(input_data, output_data)
        
        # Verify peak is detected
        peaks_detected = np.sum(output_data[0][500:510])
        self.assertGreater(peaks_detected, 5)  # At least 5 samples detected

if __name__ == '__main__':
    # Unit tests
    unittest.main(argv=[''], exit=False, verbosity=2)
    
    # Performance tests
    import time
    detector = peak_detector()
    
    # Throughput test
    large_signal = np.random.randn(1000000)
    input_data = [large_signal]
    output_data = [np.zeros(1000000)]
    
    start_time = time.time()
    detector.work(input_data, output_data)
    elapsed = time.time() - start_time
    
    throughput = len(large_signal) / elapsed / 1e6
    print(f"\nThroughput: {throughput:.2f} MSps")
    print(f"Processing time: {elapsed*1000:.1f} ms")

      Performance Comparison

      Performance Metrics by Approach

      Approach Throughput Memory Dev. Time Maintenance
      Pure Python Low High Fast Easy
      Vectorized NumPy Medium Medium Medium Easy
      Standard C++ High Low Slow Complex
      C++ SIMD Very High Very Low Very Slow Very Complex
      Hierarchical Inherited Variable Very Fast Very Easy

      Recommendations by Use Case

      Rapid prototyping: Python + NumPy Low-rate production: Optimized Python High-rate production: C++ Simple reuse: Hierarchical blocks

      • Start with Python for prototyping
      • Switch to C++ only if necessary for performance
      • Always test and document
      • Consider long-term maintenance
