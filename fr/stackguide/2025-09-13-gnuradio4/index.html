      Créer ses propres blocs GNU Radio

      Types de blocs dans GNU Radio

      Classification par fonctionnalité

      GNU Radio organise les blocs en plusieurs catégories selon leur comportement :

      Sync Blocks : Ratio d’entrée/sortie fixe 1:1

      • Un échantillon d’entrée → Un échantillon de sortie
      • Exemple : Multiply, Add, Complex to Mag
      • Le plus simple à implémenter

      General Blocks : Ratio d’entrée/sortie variable

      • Nombre d’échantillons consommés/produits peut varier
      • Exemple : Decimator, Interpolator, Frame Sync
      • Plus complexe mais très puissant

      Hierarchical Blocks : Composition de blocs existants

      • Encapsule plusieurs blocs en un seul
      • Réutilisabilité et simplicité d’interface
      • Exemple : Modulateur complet, Filtre composite

      Source/Sink Blocks : Interface avec l’extérieur

      • Sources : Génèrent des données
      • Sinks : Consomment des données
      • Interface matérielle ou fichiers

      Choix du type selon l’usage

      Type Cas d’usage Complexité Performance
      Sync Opérations mathématiques simples Faible Élevée
      General Changement de débit, protocoles Élevée Variable
      Hierarchical Réutilisation, interface simple Moyenne Héritée
      Source/Sink Interface matérielle Élevée Critique

      Quand créer un bloc personnalisé

      Signaux d’alarme pour un bloc custom

      Flowgraph trop complexe :

      • Plus de 20 blocs pour une fonction simple
      • Répétition du même sous-ensemble de blocs
      • Difficulté à maintenir ou modifier

      Performance insuffisante :

      • Opérations répétitives en Python pur
      • Goulots d’étranglement identifiés
      • Latence trop élevée pour l’application

      Fonctionnalité manquante :

      • Algorithme propriétaire ou spécialisé
      • Interface avec matériel spécifique
      • Protocole de communication non supporté

      Alternatives à considérer d’abord

      Avant de créer un bloc, vérifiez :

      1. gr- modules existants* : Plus de 50 modules tiers disponibles
      2. Combination de blocs : Hierarchical block peut suffire
      3. Scripts Python externes : File Source/Sink pour interface

      Création d’un bloc Python simple

      Exemple : Détecteur de crête adaptatif

      Créons un bloc qui détecte les pics dans un signal avec un seuil adaptatif.

      Étape 1 : Structure de base

      Créez un nouveau fichier peak_detector.py :

      #!/usr/bin/env python3

import numpy as np
from gnuradio import gr

class peak_detector(gr.sync_block):
    """
    Détecteur de crête adaptatif
    """
    def __init__(self, threshold_ratio=0.8, adaptation_rate=0.01):
        # Définition des ports d'entrée/sortie
        gr.sync_block.__init__(
            self,
            name="peak_detector",
            in_sig=[np.float32],    # Une entrée float
            out_sig=[np.float32]    # Une sortie float
        )
        
        # Paramètres du bloc
        self.threshold_ratio = threshold_ratio
        self.adaptation_rate = adaptation_rate
        
        # Variables d'état internes
        self.peak_level = 0.0
        self.threshold = 0.0

    def work(self, input_items, output_items):
        """
        Fonction principale de traitement
        """
        in0 = input_items[0]    # Signal d'entrée
        out = output_items[0]   # Signal de sortie
        
        # Traitement échantillon par échantillon
        for i in range(len(in0)):
            sample = abs(in0[i])
            
            # Adaptation du seuil
            if sample > self.peak_level:
                self.peak_level = sample
            else:
                # Décroissance lente du pic
                self.peak_level *= (1 - self.adaptation_rate)
            
            # Calcul du seuil
            self.threshold = self.peak_level * self.threshold_ratio
            
            # Détection de pic
            if sample > self.threshold:
                out[i] = 1.0  # Pic détecté
            else:
                out[i] = 0.0  # Pas de pic
        
        # Retourne le nombre d'échantillons traités
        return len(output_items[0])

      Améliorons le bloc avec des paramètres modifiables :

    gr.sync_block.__init__(

# Getters/Setters pour GNU Radio Companion
def threshold_ratio(self):
    return self._threshold_ratio

def set_threshold_ratio(self, threshold_ratio):
    self._threshold_ratio = threshold_ratio

def adaptation_rate(self):
    return self._adaptation_rate

def set_adaptation_rate(self, adaptation_rate):
    self._adaptation_rate = adaptation_rate

      Étape 3 : Test unitaire du bloc

      Créez test_peak_detector.py :

      #!/usr/bin/env python3

import numpy as np
import matplotlib.pyplot as plt
from peak_detector import peak_detector

def test_peak_detector():
    # Génération d'un signal de test
    t = np.linspace(0, 1, 1000)
    signal = np.sin(2 * np.pi * 5 * t)  # 5 Hz
    
    # Ajout de pics artificiels
    signal[200:210] += 2.0  # Pic à t=0.2s
    signal[500:510] += 1.5  # Pic à t=0.5s
    signal[800:820] += 3.0  # Pic plus large à t=0.8s
    
    # Instanciation du bloc
    detector = peak_detector(threshold_ratio=0.7, adaptation_rate=0.05)
    
    # Simulation du traitement
    output = np.zeros_like(signal)
    
    # Traitement par blocs (simulation GNU Radio)
    block_size = 100
    for i in range(0, len(signal), block_size):
        end = min(i + block_size, len(signal))
        input_block = [signal[i:end]]
        output_block = [output[i:end]]
        
        detector.work(input_block, output_block)
    
    # Visualisation
    plt.figure(figsize=(12, 6))
    plt.subplot(2, 1, 1)
    plt.plot(t, signal)
    plt.title('Signal d\'entrée avec pics')
    plt.ylabel('Amplitude')
    
    plt.subplot(2, 1, 2)
    plt.plot(t, output)
    plt.title('Sortie du détecteur de pics')
    plt.ylabel('Détection (1=pic)')
    plt.xlabel('Temps (s)')
    
    plt.tight_layout()
    plt.show()
    
    print(f"Pics détectés : {np.sum(output)} échantillons")

if __name__ == '__main__':
    test_peak_detector()

      Optimisation et blocs C++

      Quand passer au C++

      • Latence critique

      Exemple de portage C++

      Structure de base pour le même détecteur de pic :

      // peak_detector_impl.h
#include <gnuradio/sync_block.h>

class peak_detector_impl : public gr::sync_block
{
private:
    float d_threshold_ratio;
    float d_adaptation_rate;
    float d_peak_level;
    float d_threshold;

public:
    peak_detector_impl(float threshold_ratio, float adaptation_rate);
    ~peak_detector_impl();

    int work(int noutput_items,
             gr_vector_const_void_star &input_items,
             gr_vector_void_star &output_items) override;

    void set_threshold_ratio(float threshold_ratio);
    float threshold_ratio() const;
};
      // peak_detector_impl.cc
int peak_detector_impl::work(int noutput_items,
                           gr_vector_const_void_star &input_items,
                           gr_vector_void_star &output_items)
{
    const float *in = (const float *) input_items[0];
    float *out = (float *) output_items[0];

    // Boucle optimisée
    for (int i = 0; i < noutput_items; i++) {
        float sample = std::abs(in[i]);
        
        // Adaptation vectorisée si possible
        if (sample > d_peak_level) {
            d_peak_level = sample;
        } else {
            d_peak_level *= (1.0f - d_adaptation_rate);
        }
        
        d_threshold = d_peak_level * d_threshold_ratio;
        out[i] = (sample > d_threshold) ? 1.0f : 0.0f;
    }

    return noutput_items;
}

      Tests et validation

      Suite de tests complète 

      #!/usr/bin/env python3

import unittest
import numpy as np
from peak_detector import peak_detector

class TestPeakDetector(unittest.TestCase):
    
    def setUp(self):
        self.detector = peak_detector(threshold_ratio=0.8, adaptation_rate=0.1)
    
    def test_no_input(self):
        """Test avec signal nul"""
        input_data = [np.zeros(100)]
        output_data = [np.zeros(100)]
        
        result = self.detector.work(input_data, output_data)
        
        self.assertEqual(result, 100)
        self.assertTrue(np.all(output_data[0] == 0))
    
    def test_constant_signal(self):
        """Test avec signal constant"""
        input_data = [np.ones(100)]
        output_data = [np.zeros(100)]
        
        self.detector.work(input_data, output_data)
        
        # Le premier échantillon devrait être détecté
        self.assertEqual(output_data[0][0], 1.0)
    
    def test_parameter_bounds(self):
        """Test des limites de paramètres"""
        # Test threshold_ratio
        self.detector.set_threshold_ratio(0.5)
        self.assertEqual(self.detector.threshold_ratio(), 0.5)
        
        # Test adaptation_rate  
        self.detector.set_adaptation_rate(0.01)
        self.assertEqual(self.detector.adaptation_rate(), 0.01)
    
    def test_peak_detection_accuracy(self):
        """Test précision de détection"""
        # Signal avec pic connu
        signal = np.zeros(1000)
        signal[500:510] = 2.0  # Pic de 10 échantillons
        
        input_data = [signal]
        output_data = [np.zeros(1000)]
        
        self.detector.work(input_data, output_data)
        
        # Vérifier que le pic est détecté
        peaks_detected = np.sum(output_data[0][500:510])
        self.assertGreater(peaks_detected, 5)  # Au moins 5 échantillons détectés

if __name__ == '__main__':
    # Tests unitaires
    unittest.main(argv=[''], exit=False, verbosity=2)
    
    # Tests de performance
    import time
    detector = peak_detector()
    
    # Test de débit
    large_signal = np.random.randn(1000000)
    input_data = [large_signal]
    output_data = [np.zeros(1000000)]
    
    start_time = time.time()
    detector.work(input_data, output_data)
    elapsed = time.time() - start_time
    
    throughput = len(large_signal) / elapsed / 1e6
    print(f"\nDébit : {throughput:.2f} MSps")
    print(f"Temps de traitement : {elapsed*1000:.1f} ms")

      Conclusion

      La création de blocs personnalisés transforme GNU Radio d’outil de prototypage en plateforme de développement complète. Cette compétence ouvre la porte à :

      • Implémentation d’algorithmes propriétaires
      • Optimisation de performances critiques
      • Interface avec du matériel spécialisé
      • Contribution à la communauté open source

      Points clés à retenir :

      • Commencer par Python pour le prototypage
      • Passer au C++ uniquement si nécessaire pour les performances
      • Toujours tester et documenter
      • Considérer la maintenance à long terme
