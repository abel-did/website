      Premier contact avec GNU Radio Companion

      10 septembre 2025·Temps de lecture : 7 minutes

      Après avoir installé GNU Radio, il est temps de découvrir son interface graphique principale : GNU Radio Companion (GRC). Cette interface permet de créer des chaînes de traitement de signal par glisser-déposer, rendant accessible la puissance de GNU Radio sans programmation directe.

      Interface de GNU Radio Companion

      Lancement et première impression 

      # Lancer GNU Radio Companion
gnuradio-companion

# Ou depuis le menu Applications (selon la distribution)
# Applications → Engineering → GNU Radio Companion

      L’interface de GRC se compose de quatre zones principales :

      1. Barre d’outils : Nouveau, Ouvrir, Sauvegarder, Exécuter, Arrêter
      2. Palette de blocs (gauche) : Catalogue des blocs disponibles, organisés par catégorie
      3. Canevas central : Zone de travail pour assembler le flowgraph
      4. Console (bas) : Messages d’erreur, avertissements, et sortie du programme

      Organisation de la palette de blocs

      Les blocs sont organisés en catégories logiques :

      • Core : Blocs fondamentaux (sources, sinks, opérateurs)
      • Filters : Filtres FIR, IIR, décimateurs, interpolateurs
      • Math Operations : Opérations mathématiques, FFT, corrélation
      • Modulators/Demodulators : AM, FM, PSK, QAM, etc.
      • GUI : Interfaces graphiques (scopes, spectres, contrôles)
      • Audio : Sources et sinks audio système
      • Blocks : Blocs génériques et utilitaires

      Test pratique : Comptez le nombre de blocs disponibles dans votre installation. Une installation complète GNU Radio 3.10+ contient plus de 200 blocs de base.

      Concepts fondamentaux : blocs et connexions

      Les blocs : unités de base

      Chaque bloc dans GNU Radio représente une fonction de traitement de signal :

      Sources : Génèrent des données

      • Signal Source : Génère sinusoïdes, carrés, triangles
      • Noise Source : Bruit blanc, gaussien
      • File Source : Lit des fichiers de données

      Sinks : Consomment des données

      • QT GUI Time Sink : Oscilloscope temporel
      • QT GUI Frequency Sink : Analyseur de spectre
      • File Sink : Écrit dans un fichier

      Processing blocks : Transforment les données

      • Multiply : Multiplication de signaux
      • Add : Addition de signaux
      • Low Pass Filter : Filtre passe-bas

      Types de connexions

      GNU Radio utilise un système de types stricts :

      • Complex (bleu) : Données complexes I/Q
      • Float (orange) : Nombres flottants réels
      • Int (jaune) : Nombres entiers
      • Byte (vert) : Données 8 bits

      Règle importante : Vous ne pouvez connecter que des ports de même type. GRC indique les erreurs de type par des connexions rouges.

      Premier flowgraph simple

      Générateur de signal basique

      Créons un premier flowgraph pour générer et visualiser un signal :

      1. Ajout des blocs :

        • Glissez Signal Source depuis Core → Sources
        • Ajoutez Throttle depuis Core → Misc
        • Ajoutez QT GUI Time Sink depuis Core → GUI

      2. Configuration des paramètres :

        Signal Source :

        • Output Type : Complex
        • Sample Rate : 32000
        • Waveform : Cosine
        • Frequency : 1000
        • Amplitude : 1

        Throttle :

        • Type : Complex
        • Sample Rate : 32000

        QT GUI Time Sink :

        • Type : Complex
        • Sample Rate : 32000
        • Number of Points : 1024

      3. Connexions :

        Signal Source → Throttle → QT GUI Time Sink

      Analyse du flowgraph

      Ce flowgraph simple illustre les concepts clés :

      • Signal Source génère une sinusoïde complexe à 1 kHz
      • Throttle limite le débit pour éviter la surcharge CPU
      • Time Sink affiche le signal en temps réel

      Pourquoi le Throttle ? Sans lui, GNU Radio générerait des données aussi vite que possible, saturant le CPU. Le Throttle simule une fréquence d’échantillonnage réaliste.

      Types de données et fréquences d’échantillonnage

      Correspondance des types

      Type GRC Description Taille Usage typique
      complex Complexe I/Q 8 bytes Signaux RF, bande de base
      float Flottant 32-bit 4 bytes Audio, signaux réels
      int Entier 32-bit 4 bytes Compteurs, contrôle
      byte Entier 8-bit 1 byte Données binaires

      Gestion des fréquences d’échantillonnage

      Règle d’or : Tous les blocs d’un chemin doivent avoir la même fréquence d’échantillonnage, sauf utilisation explicite de décimateurs/interpolateurs.

      Exemple de calcul de débit :

      • Signal complexe à 2 MSps = 2×10⁶ × 8 bytes = 16 MB/s
      • Signal float à 48 kSps = 48×10³ × 4 bytes = 192 kB/s

      Test pratique : détection d’erreur de débit

      Créez intentionnellement une erreur :

      1. Signal Source (32 kSps) → Time Sink (48 kSps)
      2. Exécutez le flowgraph
      3. Observez l’erreur dans la console : 
        RuntimeError: check topology failed: input multiple different sample rates

      Exécution et débogage

      Modes d’exécution

      GRC offre plusieurs modes d’exécution :

      Mode normal (F5) :

      • Génère le code Python
      • Compile si nécessaire
      • Lance l’exécution

      Mode pas-à-pas :

      • Generate (F7) : Génère le code seulement
      • Execute (F6) : Lance le code généré

      Messages d’erreur courants

      1. Erreur de connexion :

      RuntimeError: cannot connect output port X to input port Y

      Solution : Vérifiez les types de données (couleurs des connexions)

      2. Paramètre manquant :

      Missing required parameter: Sample Rate

      Solution : Remplissez tous les champs obligatoires (en rouge)

      3. Flowgraph non connecté :

      Flow graph has unconnected blocks

      Solution : Tous les blocs doivent être connectés dans un chemin cohérent

      Console de débogage

      La console affiche des informations utiles :

      • INFO : Informations générales sur l’exécution
      • WARNING : Avertissements non bloquants
      • ERROR : Erreurs empêchant l’exécution
      • DEBUG : Informations détaillées (mode verbose)

      Génération de code Python

      Code généré automatiquement

      GRC génère automatiquement du code Python équivalent. Pour un flowgraph simple :

      #!/usr/bin/env python3
# -*- coding: utf-8 -*-

from gnuradio import blocks
from gnuradio import gr
from gnuradio import qtgui
from gnuradio.filter import firdes

class simple_flowgraph(gr.top_block):
    def __init__(self):
        gr.top_block.__init__(self, "Simple Signal Generator")
        
        # Blocks
        self.signal_source = blocks.signal_source_c(
            32000,          # Sample rate
            blocks.GR_COS_WAVE,  # Waveform
            1000,           # Frequency
            1,              # Amplitude
            0               # Offset
        )
        
        self.throttle = blocks.throttle(
            gr.sizeof_gr_complex*1, 
            32000,
            True
        )
        
        self.qtgui_time_sink = qtgui.time_sink_c(
            1024,           # Size
            32000,          # Sample rate
            "",             # Name
            1               # Number of inputs
        )
        
        # Connections
        self.connect((self.signal_source, 0), (self.throttle, 0))
        self.connect((self.throttle, 0), (self.qtgui_time_sink, 0))

      Avantages de l’approche visuelle

      Pour l’apprentissage :

      • Visualisation immédiate du flux de données
      • Paramètres accessibles par interface graphique
      • Détection d’erreur en temps réel

      Pour le prototypage :

      • Modification rapide de paramètres
      • Test de différentes configurations
      • Sauvegarde de variantes

      Comparaison avec d’autres outils visuels

      GNU Radio Companion vs alternatives

      Outil Type Avantages Inconvénients
      GRC Open source Gratuit, extensible, communauté active Courbe d’apprentissage
      MATLAB Simulink Propriétaire Interface soignée, intégration MATLAB Coût, lock-in
      LabVIEW Propriétaire Excellent pour l’instrumentation Très cher, spécialisé
      SDR# Plugins Gratuit/limité Facile pour la réception Limité, Windows uniquement

      Forces spécifiques de GRC

      1. Écosystème complet : Plus de 200 blocs de base
      2. Code ouvert : Possibilité de modifier/étendre
      3. Multi-plateforme : Linux, Windows, macOS
      4. Génération de code : Transition facile vers Python pur
      5. Communauté active : Blocs tiers, exemples, support

      Cas d’usage optimaux

      GRC excelle pour :

      • Prototypage rapide d’algorithmes SDR
      • Éducation et formation
      • Test de concepts avant implémentation
      • Applications temps réel simples/moyennes

      Limites de GRC :

      • Performances limitées pour applications critiques
      • Interface parfois lente avec flowgraphs complexes
      • Débogage difficile sur gros projets

      Exercices pratiques

      Exercice 1 : Signal multi-fréquences

      Créez un générateur de deux sinusoïdes (1 kHz et 3 kHz) :

      1. Deux Signal Source
      2. Un bloc Add pour les combiner
      3. Visualisation sur Time Sink et Frequency Sink

      Exercice 2 : Analyse spectrale

      Ajoutez un analyseur de spectre à votre premier flowgraph :

      1. Connectez le Throttle vers QT GUI Frequency Sink
      2. Observez le pic à 1 kHz
      3. Modifiez la fréquence du générateur, observez le changement

      Exercice 3 : Modulation simple

      Implémentez une modulation d’amplitude basique :

      1. Signal porteur (10 kHz)
      2. Signal modulant (1 kHz)
      3. Multiplieur pour la modulation
      4. Visualisation temporelle et fréquentielle

      Conclusion

      GNU Radio Companion fournit une introduction accessible au monde de la SDR. Son approche visuelle permet de comprendre rapidement les concepts de flux de données et de traitement de signal. Bien que limité pour des applications de production, GRC reste un excellent outil de prototypage et d’apprentissage.

      Dans le prochain article, nous utiliserons ces bases pour créer notre premier modulateur AM complet, introduisant des concepts plus avancés de modulation et de génération de signaux.
