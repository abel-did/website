      Créer un modulateur AM avec GNU Radio

      10 septembre 2025·Temps de lecture : 7 minutes

      Après avoir découvert GNU Radio Companion, il est temps de créer notre premier projet concret : un modulateur AM (Amplitude Modulation). Ce guide vous accompagne step-by-step pour comprendre, implémenter et tester un modulateur d’amplitude fonctionnel.

      Rappels théoriques sur la modulation AM

      Principe de base

      La modulation d’amplitude (AM) consiste à faire varier l’amplitude d’un signal porteur en fonction du signal d’information à transmettre. Mathématiquement, un signal AM s’exprime :

      sAM(t)=Ac[1+m⋅sm(t)]cos⁡(2πfct)s_{AM}(t) = A_c[1 + m \cdot s_m(t)] \cos(2\pi f_c t)sAM​(t)=Ac​[1+m⋅sm​(t)]cos(2πfc​t)

      Où :

      • $A_c$ : Amplitude de la porteuse
      • $m$ : Indice de modulation (0 ≤ m ≤ 1)
      • $s_m(t)$ : Signal modulant (audio)
      • $f_c$ : Fréquence de la porteuse

      Paramètres critiques

      Indice de modulation (m) :

      • m = 0 : Pas de modulation (porteuse pure)
      • m = 1 : Modulation 100% (maximum sans distorsion)
      • m > 1 : Surmodulation (distorsion)

      Bande passante : La bande passante d’un signal AM est : $BW = 2 \times f_{max}$ Pour un signal audio jusqu’à 5 kHz : BW = 10 kHz

      Avantages et inconvénients

      Avantages :

      • Simplicité de démodulation
      • Récepteurs peu coûteux
      • Robustesse aux interférences de phase

      Inconvénients :

      • Efficacité énergétique faible (25% théorique)
      • Sensible au bruit d’amplitude
      • Occupation spectrale importante

      Architecture du modulateur dans GRC

      Vue d’ensemble du flowgraph

      Notre modulateur AM comprendra :

      1. Source audio : Signal modulant (musique, voix)
      2. Générateur de porteuse : Signal sinusoïdal à la fréquence désirée
      3. Bloc de modulation : Multiplication et addition
      4. Contrôles : Réglage de l’indice de modulation
      5. Visualisation : Scopes temporel et fréquentiel
      6. Sortie : Fichier ou transmission RF

                 ├─── [Multiply] ──┐

      Implémentation step-by-step

      Étape 1 : Sources et générateurs

      1.1 Source audio

      • Bloc : Audio Source
      • Sample Rate : 48000
      • Device Name : (défaut système)

      1.2 Générateur de porteuse

      • Bloc : Signal Source
      • Sample Rate : 48000
      • Waveform : Cosine
      • Frequency : 10000 (10 kHz pour test)
      • Amplitude : 1

      1.3 Contrôle d’indice de modulation

      • Bloc : QT GUI Range
      • ID : mod_index
      • Default Value : 0.5
      • Minimum : 0
      • Maximum : 1
      • Step : 0.01

      Étape 2 : Blocs de traitement

      2.1 Multiplication pour la modulation

      • Bloc : Multiply
      • Connecter : Audio Source × mod_index

      2.2 Addition de la porteuse

      • Bloc : Add
      • Connecter : (Audio × mod_index) + Carrier

      2.3 Contrôle d’amplitude finale

      • Bloc : Multiply Const
      • Constant : 0.5 (pour éviter la saturation)

      Étape 3 : Visualisation et sortie

      3.1 Scope temporel

      • Bloc : QT GUI Time Sink
      • Sample Rate : 48000
      • Number of Points : 2048
      • Connecter : Sortie du modulateur

      3.2 Analyseur de spectre

      • Bloc : QT GUI Frequency Sink
      • Sample Rate : 48000
      • FFT Size : 2048
      • Connecter : Sortie du modulateur

      3.3 Sortie audio

      • Bloc : Audio Sink
      • Sample Rate : 48000
      • Connecter : Sortie du modulateur

      Configuration complète des paramètres

      Bloc Paramètre Valeur Justification
      Audio Source Sample Rate 48000 Standard audio
      Signal Source Frequency 10000 Audible pour test
      QT GUI Range Maximum 1.0 Éviter surmodulation
      Time Sink Points 2048 Résolution temporelle
      FFT Sink Size 2048 Résolution fréquentielle

      Tests et validation du modulateur

      Test 1 : Signal sinusoïdal

      Remplacez temporairement l’Audio Source par un Signal Source :

      • Frequency : 1000 Hz
      • Amplitude : 1

      Résultat attendu :

      • Scope temporel : Porteuse avec enveloppe sinusoïdale
      • Scope fréquentiel : Pic à 10 kHz ± 1 kHz (bandes latérales)

      Test 2 : Vérification de l’indice de modulation

      Procédure :

      1. Réglez mod_index à 0 → Porteuse pure visible
      2. Augmentez à 0.5 → Modulation visible, pas de distorsion
      3. Montez à 1.0 → Modulation maximale acceptable
      4. Dépassez 1.0 → Distorsion visible (surmodulation)

      Mesures pratiques :

      • À m = 0.5 : Profondeur de modulation = 50%
      • À m = 1.0 : L’enveloppe touche zéro sans le dépasser

      Test 3 : Analyse spectrale

      Signal audio complexe :

      • Utilisez une source audio réelle (musique)
      • Observez l’étalement spectral autour de 10 kHz
      • Vérifiez la symétrie des bandes latérales

      Métriques de qualité :

      • THD (Total Harmonic Distortion) < 1% pour m ≤ 0.8
      • Rapport signal/bruit > 40 dB
      • Suppression de la porteuse résiduelle

      Script de test automatisé 

      #!/usr/bin/env python3
import numpy as np
from gnuradio import gr, blocks, analog

class AM_Tester(gr.top_block):
    def __init__(self):
        gr.top_block.__init__(self)
        
        # Test signal
        self.sig_source = analog.sig_source_f(
            48000, analog.GR_SIN_WAVE, 1000, 1, 0)
        
        # AM modulator
        self.carrier = analog.sig_source_c(
            48000, analog.GR_COS_WAVE, 10000, 1, 0)
        
        self.multiply = blocks.multiply_ff()
        self.add = blocks.add_cc()
        
        # Connections
        self.connect(self.sig_source, (self.multiply, 0))
        self.connect(self.carrier, (self.add, 1))

      Optimisations et variantes

      Amélioration 1 : Filtrage audio

      Ajoutez un filtre passe-bas pour limiter la bande audio :

      Filtre audio :

      • Bloc : Low Pass Filter
      • Sample Rate : 48000
      • Cutoff Freq : 5000
      • Transition Width : 1000

      Amélioration 2 : AGC (Automatic Gain Control)

      Pour maintenir un niveau de modulation constant :

      AGC Block :

      • Bloc : AGC
      • Rate : 1e-3
      • Reference : 1.0
      • Gain : 1.0

      Amélioration 3 : Double Sideband Suppressed Carrier (DSB-SC)

      Supprimez la porteuse pour améliorer l’efficacité :

      Modification :

      • Retirez l’addition de la porteuse
      • Gardez seulement la multiplication
      • Résultat : signal DSB-SC

      Variante : AM synchrone

      Pour améliorer la qualité de démodulation :

      Ajouts :

      • Pilote de synchronisation à faible niveau
      • Filtrages sélectifs
      • Contrôle automatique de fréquence

      Comparaison avec d’autres techniques

      AM vs FM

      Critère AM FM
      Efficacité énergétique 25% 100%
      Complexité récepteur Simple Complexe
      Qualité audio Moyenne Excellente
      Résistance au bruit Faible Élevée
      Bande passante 2×f_max 2×(Δf + f_max)

      AM vs Modulations numériques

      Avantages de l’AM :

      • Démodulation simple (détecteur d’enveloppe)
      • Compatible avec récepteurs existants
      • Coût réduit

      Inconvénients face au numérique :

      • Efficacité spectrale faible
      • Pas de correction d’erreur
      • Qualité limitée

      Tests comparatifs sur signal audio 1 kHz :

      DSB-SC 45 0.5 50
      SSB 48 0.3 100

      Applications pratiques et limitations

      Applications réelles

      Broadcasting AM :

      • Radio AM commerciale (530-1600 kHz)
      • Radio maritime (2-30 MHz)
      • Aviation (108-137 MHz)

      Applications industrielles :

      • Transmissions de données faible débit
      • Télémétrie simple
      • Communications d’urgence

      Limitations techniques

      Bande passante : Notre modulateur 48 kSps limite les applications RF réelles. Pour du vrai RF :

      • Utilisez des sample rates > 1 MSps
      • Ajoutez des filtres de reconstruction
      • Intégrez un upconverter RF

      Dynamique :

      • 16 bits limitent le rapport signal/bruit
      • Utilisez des ADC/DAC haute résolution pour la production

      Latence :

      • Buffer audio introduit ~100ms de délai
      • Optimisez pour les applications temps réel

      Améliorations pour la production

      Filtrage avancé :

      • Filtres anti-repli précis
      • Égalisation audio
      • Limiteurs de modulation

      Monitoring :

      • Mesure en temps réel de l’indice de modulation
      • Détection de surmodulation
      • Analyse spectrale continue

      Interfaçage RF :

      • Upconverter vers fréquences radio
      • Amplificateurs de puissance
      • Filtres d’émission

      Conclusion

      Ce modulateur AM démontre les concepts fondamentaux de la modulation d’amplitude tout en illustrant la puissance de GNU Radio pour le prototypage rapide. Bien que simple, cette implémentation pose les bases pour des systèmes plus complexes.

      Les points clés à retenir :

      • L’indice de modulation détermine la qualité
      • La visualisation temps/fréquence est essentielle
      • Les optimisations dépendent de l’application cible
