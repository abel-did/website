      Lage en AM-modulator med GNU Radio

      10. september 2025·Lesetid : 5 minutter

      Etter å ha oppdaget GNU Radio Companion, er det på tide å lage vårt første konkrete prosjekt: en AM (Amplitude Modulation) modulator. Denne guiden følger deg steg-for-steg for å forstå, implementere og teste en funksjonell amplitudemodulator.

      Teoretisk gjennomgang av AM-modulasjon

      Grunnprinsipp

      Amplitudemodulasjon (AM) består av å variere amplituden til et bærersignal i henhold til informasjonssignalet som skal overføres. Matematisk uttrykkes et AM-signal som:

      sAM(t)=Ac[1+m⋅sm(t)]cos⁡(2πfct)s_{AM}(t) = A_c[1 + m \cdot s_m(t)] \cos(2\pi f_c t)sAM​(t)=Ac​[1+m⋅sm​(t)]cos(2πfc​t)

      Hvor:

      • $A_c$: Bæreramplitude
      • $m$: Modulasjonsindeks (0 ≤ m ≤ 1)
      • $s_m(t)$: Modulerende signal (lyd)
      • $f_c$: Bærerfrekvens

      Kritiske parametere

      Modulasjonsindeks (m):

      • m = 0: Ingen modulasjon (ren bærer)
      • m = 1: 100% modulasjon (maksimum uten distorsjon)
      • m > 1: Overmodulasjon (distorsjon)

      Båndbredde: Båndbredden til et AM-signal er: $BW = 2 \times f_{max}$ For et lydsignal opp til 5 kHz: BW = 10 kHz

      Fordeler og ulemper

      Fordeler:

      • Enkel demodulasjon
      • Lave kostnader mottakere
      • Robust mot faseinterferens

      Ulemper:

      • Lav energieffektivitet (25% teoretisk)
      • Følsom for amplitudestøy
      • Stor spektral okkupasjon

      Modulatorarkitektur i GRC

      Flowgraph-oversikt

      Vår AM-modulator vil inkludere:

      1. Lydkilde: Modulerende signal (musikk, stemme)
      2. Bærergenerator: Sinusformet signal ved ønsket frekvens
      3. Modulasjonsblokk: Multiplikasjon og addisjon
      4. Kontroller: Modulasjonsindeks-justering
      5. Visualisering: Tids- og frekvensskop
      6. Utgang: Fil eller RF-overføring

      Funksjonelt diagram 

      [Audio Source] ──┐
                 ├─── [Multiply] ──┐
[Mod Index] ─────┘                │
                                  ├─── [Add] ──── [Output]
[Carrier] ────────────────────────┘

      Steg-for-steg implementering

      Steg 1: Kilder og generatorer

      1.1 Lydkilde

      • Blokk: Audio Source
      • Sample Rate: 48000
      • Device Name: (systemstandard)

      1.2 Bærergenerator

      • Blokk: Signal Source
      • Sample Rate: 48000
      • Waveform: Cosine
      • Frequency: 10000 (10 kHz for testing)
      • Amplitude: 1

      1.3 Modulasjonsindeks kontroll

      • Blokk: QT GUI Range
      • ID: mod_index
      • Default Value: 0.5
      • Minimum: 0
      • Maximum: 1
      • Step: 0.01

      Steg 2: Behandlingsblokker

      2.1 Multiplikasjon for modulasjon

      • Blokk: Multiply
      • Koble: Audio Source × mod_index

      2.2 Bæreraddisjon

      • Blokk: Add
      • Koble: (Audio × mod_index) + Carrier

      2.3 Final amplitudekontroll

      • Blokk: Multiply Const
      • Constant: 0.5 (for å unngå metning)

      Steg 3: Visualisering og utgang

      3.1 Tidsskop

      • Blokk: QT GUI Time Sink
      • Sample Rate: 48000
      • Number of Points: 2048
      • Koble: Modulatorutgang

      3.2 Spektrumanalysator

      • Blokk: QT GUI Frequency Sink
      • Sample Rate: 48000
      • FFT Size: 2048
      • Koble: Modulatorutgang

      3.3 Lydutgang

      • Blokk: Audio Sink
      • Sample Rate: 48000
      • Koble: Modulatorutgang

      Komplett parameterkonfigurasjon

      Blokk Parameter Verdi Begrunnelse
      Audio Source Sample Rate 48000 Lydstandard
      Signal Source Frequency 10000 Hørbar for testing
      QT GUI Range Maximum 1.0 Unngå overmodulasjon
      Time Sink Points 2048 Temporal oppløsning
      FFT Sink Size 2048 Frekvensoppløsning

      Testing og validering av modulatoren

      Test 1: Sinusformet signal

      Erstatt midlertidig Audio Source med en Signal Source:

      • Frequency: 1000 Hz
      • Amplitude: 1

      Forventet resultat:

      • Tidsskop: Bærer med sinusformet enveloppe
      • Frekvensskop: Topp ved 10 kHz ± 1 kHz (sidebånd)

      Test 2: Modulasjonsindeks verifisering

      Prosedyre:

      1. Sett mod_index til 0 → Ren bærer synlig
      2. Øk til 0.5 → Synlig modulasjon, ingen distorsjon
      3. Stig til 1.0 → Maksimal akseptabel modulasjon
      4. Overstig 1.0 → Synlig distorsjon (overmodulasjon)

      Praktiske målinger:

      • Ved m = 0.5: Modulasjonsdybde = 50%
      • Ved m = 1.0: Enveloppe berører null uten å overskride

      Test 3: Spektral analyse

      Komplekst lydsignal:

      • Bruk ekte lydkilde (musikk)
      • Observer spektral spredning rundt 10 kHz
      • Verifiser sidebåndsymmetri

      Kvalitetsmålinger:

      • THD (Total Harmonic Distortion) < 1% for m ≤ 0.8
      • Signal-til-støy-forhold > 40 dB
      • Restbærerundertrykkelse

      Automatisert testskript 

      #!/usr/bin/env python3
import numpy as np
from gnuradio import gr, blocks, analog

class AM_Tester(gr.top_block):
    def __init__(self):
        gr.top_block.__init__(self)
        
        # Testsignal
        self.sig_source = analog.sig_source_f(
            48000, analog.GR_SIN_WAVE, 1000, 1, 0)
        
        # AM modulator
        self.carrier = analog.sig_source_c(
            48000, analog.GR_COS_WAVE, 10000, 1, 0)
        
        self.multiply = blocks.multiply_ff()
        self.add = blocks.add_cc()
        
        # Forbindelser
        self.connect(self.sig_source, (self.multiply, 0))
        self.connect(self.carrier, (self.add, 1))
        
    def measure_modulation_depth(self):
        # Modulasjonsdybde målingskode
        pass

if __name__ == '__main__':
    tb = AM_Tester()
    tb.start()
    # Automatiske målinger
    tb.stop()

      Optimaliseringer og varianter

      Forbedring 1: Lydfiltrering

      Legg til et lavpassfilter for å begrense lydbåndet:

      Lydfilter:

      • Blokk: Low Pass Filter
      • Sample Rate: 48000
      • Cutoff Freq: 5000
      • Transition Width: 1000

      Forbedring 2: AGC (Automatic Gain Control)

      For å opprettholde konstant modulasjonsnivå:

      AGC Block:

      • Blokk: AGC
      • Rate: 1e-3
      • Reference: 1.0
      • Gain: 1.0

      Forbedring 3: Double Sideband Suppressed Carrier (DSB-SC)

      Fjern bærer for å forbedre effektiviteten:

      Modifikasjon:

      • Fjern bæreraddisjon
      • Behold kun multiplikasjon
      • Resultat: DSB-SC signal

      Variant: Synkron AM

      For å forbedre demodulasjonskvalitet:

      Tillegg:

      • Synkroniseringspilot på lavt nivå
      • Selektiv filtrering
      • Automatisk frekvensskontroll

      Sammenligning med andre teknikker

      AM vs FM

      Kriteria AM FM
      Energieffektivitet 25% 100%
      Mottakerkompleksitet Enkel Kompleks
      Lydkvalitet Gjennomsnittlig Utmerket
      Støyresistens Lav Høy
      Båndbredde 2×f_max 2×(Δf + f_max)

      AM vs Digitale modulasjoner

      AM fordeler:

      • Enkel demodulasjon (enveloppedetektor)
      • Kompatibel med eksisterende mottakere
      • Reduserte kostnader

      Ulemper vs digitalt:

      • Lav spektral effektivitet
      • Ingen feilkorreksjon
      • Begrenset kvalitet

      Sammenlignende tester på 1 kHz lydsignal:

      DSB-SC 45 0.5 50
      SSB 48 0.3 100

      Praktiske applikasjoner og begrensninger

      Virkelige applikasjoner

      AM Broadcasting:

      • Kommersiell AM-radio (530-1600 kHz)
      • Maritim radio (2-30 MHz)
      • Luftfart (108-137 MHz)

      Industrielle applikasjoner:

      • Lav-rate dataoverføring
      • Enkel telemetri
      • Nødkommunikasjon

      Tekniske begrensninger

      Båndbredde: Vår 48 kSps modulator begrenser reelle RF-applikasjoner. For ekte RF:

      • Bruk samplingsrater > 1 MSps
      • Legg til rekonstruksjonsfiltre
      • Integrer RF-upconverter

      Dynamisk område:

      • 16 bits begrenser signal-til-støy-forhold
      • Bruk høyoppløselige ADC/DAC for produksjon

      Latens:

      • Lydbuffer introduserer ~100ms forsinkelse
      • Optimaliser for sanntidsapplikasjoner

      Produksjonsforbedringer

      Avansert filtrering:

      • Presise anti-aliasing filtre
      • Lydekvalisering
      • Modulasjonsbegrensere

      Overvåking:

      • Sanntids modulasjonsindeks måling
      • Overmodulasjonsdetektor
      • Kontinuerlig spektral analyse

      RF-grensesnitt:

      • Upconverter til radiofrekvenser
      • Effektforsterkere
      • Emisjonsfiltre
